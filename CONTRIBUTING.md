How to contribute
- Open an issue describing the bug or feature you'd like to request.
- Fork the repo, create a feature branch, make changes and open a pull request.
- Keep PRs small and focused and include tests where possible.

Local development
- Run linting: `make lint` (requires shellcheck)
- Run the tests: `make test` (this project includes a small test harness under `tests/`)

Commit messages
- Use short, descriptive commit messages and present logical units of work.

License & CLA
- By contributing you agree that your contributions will be licensed under the project's MIT license.
